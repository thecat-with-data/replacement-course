Lesson 2: Lists & Structural Hierarchy

This lesson focuses on creating and managing lists in Word, as well as understanding the importance of structural hierarchy in document organization.

By the end of this lesson, you should be able to:

  • Identify when lists should be used in professional documents
  • Create bulleted, numbered, and multi-level lists correctly
  • Represent hierarchy and structure using list levels
  • Modify and reorganize list structures efficiently
  • Avoid common list-formatting mistakes that reduce document quality

Why Lists Matter in Professional Documents

Lists are not just formatting tools - they're structural tools.

Well designed lists help readers:

  • Scan content quickly
  • Understand relationships between ideas
  • Follow steps accurately
  • Recognize hierarchy and importance

Poorly designed lists cause confusion, misinterpretation, and accessibility problems.

In professional, academic, and technical documents, lists must be clear, consistent, and intentional.

Understanding Structural Hierarchy

Hierarchy means showing relationships between information.

Examples:

  • Main idea > supporting details
  • Step > sub-step
  • Category > items

Word's list tools allow you to visually represent these relationships without manually spacing or numbering items.

Types of Lists in Word

Bulleted Lists

Use when:

  • Order does not matter
  • Items are equal in importance

Examples:

  • Features
  • Characteristics
  • Options

Professional tip: keep bullet text parallel in structure (.e.g. all start with verbs or nouns).

Numbered Lists

Use when:

  • Sequence matters
  • Instructions must be followed in order

Examples:

  1. Procedures
  2. Steps
  3. Processes

Word automatically updates numbering if you add, remove, or reorder steps. Manually typed numbers do not.

Multilevel Lists

Use when:

  • Items belong to a category
  • Steps contain sub-steps
  • An outline structure is required

Examples:

  • Project Phases
    • Planning
      • Research
      • Budgeting
    • Execution

Multilevel lists are essential for:

  • Outlines
  • Policies
  • Procedures
  • Technical documentation

Mini-Demo: Creating a Multilevel List

  1. Create a numbered or bulleted list
  2. Place the cursor on an item
  3. Use:
    • Increase Indent to demote to a sub-level
    • Decrease Indent to promote to a higher level
  4. Observe how Word changes indentation and symbols automatically.

Never use the Tab key or spaces alone to create a hierarchy.

Modifying List Appearance

Word allows you to customize lists while keeping structures intact.

You Can:

  • Change bullet styles
  • Switch between numbered and bulleted lists
  • Adjust indentation levels
  • Restart numbering (when appropriate)

Avoid:

  • Manually typing symbols (-, *, numbers)
  • Mixing list types inconsistently
  • Using spacing to fake hierarchy

Lists vs Headings

This is a common place of confusion.

Use Headings When:

  • Content represents sections of a document
  • Information will appear in a Table of Contents

Use Lists When:

  • Content represents items within a section
  • Hierarchy does not require navigation visibility

Headings define structure. Lists refine structure.

They work together, not interchangeably.

Common Mistakes

  • Manually numbering items
  • Using spaces to align list text
  • Mixing bulleted and numbered lists without purpose
  • Inconsistent indentation
  • Turning headings into lists or lists into headings

These mistakes weaken both readability and accessibility.

Accessibility Considerations

Proper list formatting is essential for screen readers.

Accessible lists:

  • Are read item-by-item in correct order
  • Preserve hierarchy for assistive technologies
  • Work correctly across devices

Improperly formatted lists (typed symbols or spacing):

  • Confuse screen readers
  • Break logical flow
  • Fail accessibility standards checks
