Lesson 3: Tables - Creation & Formatting

This lesson focuses on creating and formatting tables in Word.

By the end of this lesson, you will be able to:

  • Explain when tables should be used in Word documents
  • Create tables using multiple methods
  • Distingish between Table Design and Layout tools

  • Modify tables by adding, removing, and resizing elements
  • Avoid common mistakes that reduce readability and accesibility

Why Tables Matter in Professional Documents

Tables are data-organization tools, not visual shortcuts.

Properly constructed tables:

  • Present information clearly
  • Improve comprehension
  • Support data comparison
  • Enhance accessibility
  • Can be automatically referenced and captioned

Improper tables:

  • Confuse readers
  • Break accessibility tools
  • Cause formatting problems
  • Look unprofessional

In professional and academic documents, tables must be intentional, structured, and readable.

When to Use a Table (and When Not To)

Use a Table When:

  • Data needs to be compared
  • Information follows a row/column pattern
  • Multiple categories share attributes

Do NOT Use a Table When:

  • Aligning text visually (use tabs or alignment instead)
  • Creating page layout designs
  • For short lists or paragraphs

Rule of Thumb:

If content could be read linearly as a paragraph, a table is probably not needed.

Ways to Create Tables in Word

Method 1: Insert > Table

  • Choose the number of rows and columns visually
  • Best for simple, predictable tables

Method 2: Insert > Table > Insert Table

  • Manually specify dimensions
  • Allows for greater precision

Method 3: Convert Text to Table

  • Converts existing text into a table
  • Requires consistent delimiters (tabs or commas)

This method is useful for cleaning up poorly formatted documents.

Mini-Demo: Converting Text to a Table

  1. Select a block of text.
  2. Insert > Table > Convert Text to Table
  3. Choose how columns are separated
  4. Confirm layout and adjust as needed.

Understanding Table Tools

When a table is selected, two tool sets appear:

Table Design

Controls appearance>:

  • Table Styles
  • Header row formatting
  • Banded rows or columns
  • Borders and shading

Table Layout

Controls structure:

  • Insert/delete rows or columns
  • Cell size (height/width)
  • Merge and split cells
  • Text alignment within cells

Design = how the table looks

Layout = how the table works

Table Formatting Best Practices

Header Rows

  • Always identify column headers
  • Style them distinctly
  • Improve readability and accessibility

Consistency

  • Keep fonts, spacing, and alignment uniform
  • Avoid over-using colors

Alignment

  • Text: usually left-aligned
  • Numbers: usually right-aligned
  • Headers: centered or left-aligned

Modifying Tables

You should also be comfortable adjusting tables after creation.

Common Modifications

  • Insert rows above/below
  • Insert columns left/right
  • Resize columns or rows
  • Merge header cells for clarity
  • Split cells to separate data

Avoid

  • Dragging table edges excessively
  • Stretching tables to fit the page instead of adjusting content
  • Using manual spacing inside cells

Tables, Captions & References (Looking Ahead)

Tables often require captions in academic and professional documents.

Good tables:

  • Include a clear caption
  • Can be referenced in the text
  • Can appear automatically in a Table of Figures

You will apply captions in W-Lab 4.

Common Mistakes to Avoid

  • Using tables to align text
  • Skipping header rows
  • Mixing fonts and spacing
  • Applying too many styles
  • Making tables wider than the page
  • Using screenshots instead of actual tables

Accessibility Considerations

Accessible tables:

  • Include header rows
  • Present logical reading order
  • Avoid merged cells when possible
  • Use simple, consistent structures
