Module 3 - Lesson 3: Final Review & Professional Submission

By the end of this lesson, you will be able to:

  • Conduct a systematic final review of a Word document
  • Identify and resolve common submission errors
  • Ensure documents meet professional and academic standards
  • Perform final accessibility and privacy checks
  • Prepare clean, submission-ready files in required formats

Why Final Review Matters

Finishing a document is not the same as submitting a professional document.

In academic and workplace settings, documents are evaluated not only on content, but also on:

  • Accuracy
  • Formatting consistency
  • Accessibility
  • Clean revision history
  • Professional presentation

Many strong documents lose credibility because they were not reviewed intentionally before submission.

This lesson teaches you how to move from almost done to ready to submit.

Thinking Like a Reviewer

Before submission, shift perspective.

Ask yourself:

  • If I were the instructor/client, what would I notice first?
  • Does the document look intentional or patched together?
  • Are there signs this document went through a revision process?
  • Is anything distracting, confusing, or unfinished?

Professional writers always review documents from the reader's point of view, not just the author's.

Cleaning Up Collaboration Tools

Before submitting a final version, you must finalize collaboration elements.

Comments

All comments should be:

  • Addressed
  • Resolved
  • Removed from final submission

Leaving comments visible suggests unfinished work unless explicitly requested.

Track Changes

Determine which version is required:

  • Draft with changes (or review)
  • Clean final version (for grading/submission)

For final submissions:

  • Accept or reject all changes intentionally
  • Turn Track Changes OFF
  • Verify the document shows no markup

Review Views & Inspection

Markup Views

Toggle between:

  • All Markup - confirm revisions were addressed
  • No Markup - preview the final document

Never assume a document is clean without switching views.

Accessibility Final Check

Accessibility is not a "nice-to-have" feature—it's a professional expectation.

Run the Accessibility Checker

  • Review > Check Accessibility

Address any remaining issues:

  • Missing alt text
  • Poor heading structure
  • Low contrast warnings
  • Reading order problems

Even one unresolved issue may affect grading.

Document Inspector (Privacy & Metadata)

Metadata can expose:

  • Author names
  • Revision history
  • Comments
  • Hidden text

Final Step

  1. File > Info
  2. Check for Issues
  3. Inspect document
  4. Remove personal information if required

This step ensures privacy and professionalism.

Formatting & Consistency Review

Perform a final scan for consistency.

Check:

  • Font type and size
  • Line spacing
  • Paragraph spacing
  • Heading alignment
  • Table and figure formatting

Be alert for:

  • Random spacing
  • Inconsistent styles
  • Misaligned visuals
  • Text wrapping errors

Consistency is one of the strongest indicators of a professionally prepared document.

Final File Prep & Exporting

Follow required naming conventions exactly:

  • Lastname_Firstname_AssignmentName.docx

Exporting to PDF

When required:

  • File > Save As > PDF
  • Ensure formatting matches the Word version
  • Never submit scanned documents unless instructed.

PDFs are considered final-delivery formats.

Common Mistakes

  • Leaving Track Changes ON
  • Submitting with visible comments
  • Forgetting to update TOCs or reference tables
  • Missing alt text on visuals
  • Incorrect file names
  • Submitting draft versions by mistake

These mistakes are avoidable with a structured final review.

Final Review Checklist (Use Before Submitting)

Before submitting W-Lab 5 or the Word Project, confirm:

  • All comments resolved or removed
  • Track Changes turned OFF (unless otherwise instructed)
  • Review views checked
  • Accessibility Checker run
  • Document Inspector run
  • Formatting consistent throughout
  • File named correctly
  • Required formats submitted (DOCX, PDF if applicable)

Self-Check Questions

  1. Why is a final review different from editing?
  2. What collaboration tools must be removed before submission?
  3. Why is accessibility part of professionalism?
  4. What risks does metadata introduce?
  5. How can small formatting issues impact perception?
