Module 3 - Lesson 2: Comments, Track Changes & Document Review

By the end of this lesson, you will be able to:

  • Add and manage comments for collaborative feedback
  • Use Track Changes to record edits accurately
  • Distinguish between markup and final document views
  • Accept and reject revisions intentionally
  • Conduct a professional document review process
  • Prepare documents for final submission or delivery

Why Collaboration Tools Matter

Very few professional documents are written by one person and never revised.

In academic and workplace environments, documents are:

  • Reviewed by instructors, peers, supervisors, or clients
  • Edited in multiple stages
  • Revised based on feedback
  • Tracked for accountability

Word's collaboration tools exist to support communication, not just editing.

This lesson teaches you how to revise documents responsibly and transparently.

Using Comments for Feedback

Comments aloow reviewers to:

  • Ask questions
  • Suggest changes
  • Provide explanations
  • Flag issues without altering content

What Comments Are For

  • Feedback during drafting
  • Questions for the author
  • Notes for later revisions

What Comments Are NOT For

  • Final content
  • Long paragraphs of text
  • Permanent information

Comments should be temporary and purposeful.

Mini-Demo: Adding a Comment

  1. Highlight a word or phrase
  2. Go to Review > New Comment
  3. Type a clear, professional note

Good comments are:

  • Specific
  • Polite
  • Constructive
  • Actionable

Responding to Comments

Comments are interactive.

You can:

  • Reply to comments
  • Resolve comments
  • Reopen comments if needed

Best Practice

Resolve comments only after the issue has been addressed.

Unresolved comments should never appear in final submissions.

Track Changes: What It Does

Track Changes records edits made to a document, including:

  • Insertions
  • Deletions
  • Formatting changes

This creates a transparent revision history.

When to Use Track Changes

  • During instructor reviews
  • When collaborating with others
  • When revising drafts based on feedback

Mini-Demo: Turning on Track Changes

  1. Go to Review > Track Changes
  2. Make edits to the document
  3. Observe how changes appear as markup

Each change is labeled by author and time.

Reviewing Views: Seeing the Document Clearly

Word offers multiple review views.

Common Review Views

  • Simple Markup - clean view with revision indicators
  • All Markup - shows all changes explicitly
  • No Markup - preview of final document
  • Original - document before changes

Important Skill

You must know how to:

  • Toggle views while editing
  • Preview the final version before submission

Accepting & Rejecting Changes

Editing does not automatically finalize changes.

Accepting Changes

  • Keeps edits
  • Removes tracking

Rejecting Changes

  • Reverts to original format

Best Practice

  • Review changes one at a time
  • Never "Accept All" without review.

This ensures accuracy and accountability.

Professional Revision Etiquette

Collaboration requires professionalism.

Good Revision Practices

  • Don't delete others' comments without addressing them
  • Don't turn off Track Changes without permission or proper justification
  • Respond to feedback respectfully
  • Remember documents are often graded or audited

Revisions reflect not only skill, but professionalism.

Comparing Drafts (Conceptual Awareness)

Word allows document comparison.

While this may not be required in the lab, you should understand:

  • Drafts can be compared
  • Differences can be reviewed systematically
  • Revision history matters

This sets expectations for workplace documentation.

Common Mistakes

  • Submitting documents with Track Changes still on
  • Leaving comments unresolved
  • Accepting all changes without review
  • Turning off Track Changes too early
  • Using comments as permanent text

These mistakes are easy to avoid with practice.

Accessibility & Review Tools

Collaboration tools also support accessibility:

  • Clear revision history
  • Predictable document structure
  • Clean final output

Before any final submission:

  • All comments removed
  • Track Changes turned off
  • Final content verified

Guided Practice (Before the Lab)

Before beginning W-Lab 5, practice:

  1. Add two comments to a paragraph.
  2. Reply to one comment
  3. Turn on Track Changes
  4. Make one insertion, one deletion, and one formatting change
  5. Switch between markup views
  6. Accept one change and reject another
  7. Remove all comments and preview final document

Self-Check Questions

  2. When should Track Changes be used?
  3. Why is "Accept All" risky?
  4. What should never be included in a final submission?
  5. How do review views help during editing?
