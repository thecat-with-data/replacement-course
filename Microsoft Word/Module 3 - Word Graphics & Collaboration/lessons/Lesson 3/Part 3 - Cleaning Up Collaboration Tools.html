Part 3 - Cleaning Up Collaboration Tools

Before submitting a final version, you must finalize collaboration elements.

Comments

All comments should be:

  • Addressed
  • Resolved
  • Removed from final submission

Leaving comments visible suggests unfinished work unless explicitly requested.

Track Changes

Determine which version is required:

  • Draft with changes (or review)
  • Clean final version (for grading/submission)

For final submissions:

  • Accept or reject all changes intentionally
  • Turn Track Changes OFF
  • Verify the document shows no markup
