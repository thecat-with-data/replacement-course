Why Saving Correctly Matters

Saving a document is more than just clicking Save. It is about organization, accountability, and professionalism.

In academic and workplace environments, improper saving can cause:

  • Lost work
  • Overwritten files
  • Submission errors
  • Privacy leaks
  • Confusion during collaboration

Professionals are expected to save files in ways that make them easy to identify, retrieve, and version.

File Naming Best Practices

Unless otherwise instructed, you will use the following naming convention:

Lastname_Firstname_AssignmentName.docx

What a Good File Name Includes

    Who created it
    What it is
    Which version it is

Avoid These

  • final.docx
  • assignment.docx
  • lab2newnewFINAL.docx

These create confusion and errors, especially during grading or collaboration.

Version Control

Versioning allows you to track changes and recover from mistakes.

When to Create a New Version

  • Before major edits
  • Before revisions based on feedback
  • Before submitting a final draft

Simple Versioning System

  • _v1
  • _v2
  • _Draft
  • _Final
Example: Lastname_Firstname_W-Lab2_v2.docx

Mini-Demo: Versioning in Action

  1. Save your document normally.
  2. Use Save As to save a copy.
  3. Add _v2 to the filename.
  4. Continue editing only the new version.

By doing so, you can protect your original work by maintaining a clean version.

Exporting to PDF

PDFs preserve formatting and prevent accidental edits.

Why PDFs Are Often Required

  • Layout remains consistent
  • Fonts and spacing are preserved
  • Files are easier to view across devices
  • Final drafts look professional

How to Export to PDF

  1. File > Save As
  2. Choose PDF
  3. Confirm export

DO NOT scan or photograph documents to create PDFs.

Use the built-in export function to ensure quality and accessibility.

Sharing Documents Safely

View vs Edit Permissions

  • View: recipient can read but not make changes
  • Edit: recipient can make changes

Always check assignment instructions before sharing.

Best Practices When Sharing

  • Use view-only unless collaboration is required
  • Avoid emailing multiple versions
  • Use one shared link and update the file if needed

Inspecting Documents for Issues

Before submitting or sharing a document, you must inspect it.

What is Metadata?

Metadata is hidden information that may include:

  • Author name
  • Company name
  • Comments
  • Tracked changes
  • Hidden text

Metadata can expose private information unintentionally.

Document Inspector

What It Does

  • Finds hidden or personal data
  • Allows you to remove it safely

How to Run It

  1. File > Info
  2. Check for Issues
  3. Inspect Document

Remove personal information before submitting.

Accessibility Checker

Accessibility ensures documents are usable by everyone.

  • Missing alt text
  • Improper heading structure
  • Poor contrast
  • Long blocks of unreadable text

How to Run Accessibility Checker

  • Review > Check Accessibility

Common Mistakes

  • Sharing edit access when not required
  • Forgetting to remove track changes
  • Submitting documents with personal metadata
  • Skipping the accessibility check
  • Exporting scanned PDFs
